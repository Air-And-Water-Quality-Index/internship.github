Test Cases

  • Sensor data successfully read and uploaded to ThingSpeak
  • Alerts generated when air quality drops below safe levels
  • Responsive design tested on multiple screen sizes


Test Case 1: Sensor Connectivity

This test verifies that all environmental sensors are properly connected and transmitting accurate data to the microcontroller. It checks initial boot values and real-time readings across various environmental conditions.

Test Case 2: Cloud Data Upload

This test ensures the collected sensor data is successfully sent to ThingSpeak via Wi-Fi. The test confirms the stability of the network connection and data refresh intervals without any packet loss or delays.

Test Case 3: Alert System Functionality

The alert system is tested by simulating extreme values (e.g., high gas levels, turbidity, or temperature). This ensures the system can notify users in real-time through visual indicators and dashboard warnings.
